# Step 1 - Fundação

## Contexto

Este é meu primeiro contato real com Elixir e Phoenix LiveView.
Meu background é front-end — React, TypeScript, componentes e
estado no cliente. Para iniciar, li a documentação oficial do
Phoenix e configurei um agente de IA como mentor técnico sênior
para me guiar nas decisões arquiteturais ao longo do desafio

## O que foi feito

- Inicialização do projeto Phoenix com LiveView;
- Entendimento de estrutura de pastas;
- Decisão de mockar dados para manter o foco na interface (step 3).

## Estrutura de pastas

- Entendi a separação entre o domínio (`lib/w_core/`) e web (`lib/w_core_web/`)
- O Phoenix entrega algo parecido com React, mas com separação mais rígida entre lógica e apresentação
- w_core e w_core_web funcionam como o princípio que eu já aplico no React separando `services/` de `components/`

## Decisão de mockar os dados

- Para que os passos 1 e 2 não se tornassem um bloqueio técnico, optei pelo mock de injeção de dados, focando minha energia em garantir que a interface funcione como o esperado.
